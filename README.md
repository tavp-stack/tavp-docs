# tavp-docs

Documentation for the TAVP Stack.

## Structure (i18n-ready, per decision 10.5)

- `public/` — published HTML docs (product brief, PRD, coding standards,
  milestone checklist). These are the detailed public documents.
- `internal/` — private decision logs and notes (NOT pushed to GitHub
  public; may stay only in Gitea).
- `id/` — Indonesian guides (to be added)
- `en/` — English guides (to be added)

## Principles

- Human Style: every doc is readable by a non-programmer.
- No mention of internal build process in public docs (per INDEX.md §8).
- Keep `coding-standards.html` as the naming/style source of truth.
