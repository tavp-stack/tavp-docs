TAVP CMS v0.2.0

Content management system built on the TAVP Stack. WordPress-familiar, Voyager-style admin, thin and fast — with pluggable storage, headless API, and a modern feature set.

Version 0.2.0 (Mature) — TAVP CMS is now a credible alternative to modern CMSs like WordPress, Strapi, Statamic, and Contentful.

Quick Start #

Install

composer create-project tavp/cms my-site
cd my-site
tavp migrate
tavp serve

Configure

# .env
CMS_STORAGE=database
CMS_ADMIN_EMAILS=you@example.com
CMS_API_TOKENS=your-secret-token-here
APP_URL=https://mysite.com

Admin

Visit /admin, sign in with your allowed email via OTP.

Headless API

# List content types
curl -H "Authorization: Bearer YOUR_TOKEN" https://mysite.com/api/cms/types

# Browse posts
curl -H "Authorization: Bearer YOUR_TOKEN" https://mysite.com/api/cms/post

# Create a post
curl -X POST -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title":"Hello","slug":"hello","body":"World","status":"published"}' \
  https://mysite.com/api/cms/post

Features #

Pluggable Storage

Choose the driver in config/cms.php:

  • database — WordPress/Voyager style, via Phalcon ORM
  • flatfile — Statamic style, Markdown + YAML front matter

The rest of the CMS behaves identically regardless of driver.

BREAD Content Types

Page and Post ship by default. Define your own content types with fields and validation rules:

'content_types' => [
    'product' => [
        'label' => 'Products',
        'singular' => 'Product',
        'fields' => [
            ['name' => 'title', 'type' => 'text', 'required' => true, 'rules' => ['max:200']],
            ['name' => 'slug', 'type' => 'slug', 'from' => 'title', 'rules' => ['unique']],
            ['name' => 'price', 'type' => 'number', 'rules' => ['numeric']],
            ['name' => 'body', 'type' => 'richtext'],
            ['name' => 'status', 'type' => 'select', 'options' => ['draft', 'published']],
        ],
    ],
],

Taxonomy

Categories (hierarchical) and tags (flat) are built-in. Attach terms to any content type.

# Admin UI
/admin/taxonomy/category
/admin/taxonomy/tag

# API
GET /api/cms/taxonomy/category
POST /api/cms/taxonomy

Revisions

Every save creates a snapshot. Rollback with one click from the admin UI or API.

# API
GET /api/cms/post/1/revisions
POST /api/cms/post/1/rollback/2026-07-11_01-30-00

Search

Full-text search across all content types from admin UI and API.

# API
GET /api/cms/search?q=hello
GET /api/cms/search?q=hello&type=post

SEO & Sitemap

Per-record meta fields + auto-generated /sitemap.xml.

Webhooks

Fire HTTP POST on content created/updated/deleted with HMAC signature verification.

RBAC

Email-to-role mapping with permission patterns.

'roles' => [
    'admin@site.com' => 'admin',
    'editor@site.com' => 'editor',
],
'permissions' => [
    'admin' => ['content.*', 'taxonomy.*', 'media.*'],
    'editor' => ['content.*', 'taxonomy.*'],
],

CLI #

php bin/cms help                        # Show help
php bin/cms types                        # List content types
php bin/cms make:type product            # Create new content type
php bin/cms api:token my-app             # Generate API token
php bin/cms search:reindex               # Flush search cache
php bin/cms taxonomy:list category       # List taxonomy terms
