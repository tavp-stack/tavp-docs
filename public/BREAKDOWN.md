## 1. `prd-tavp.html` — PRD Utama

PRD utama untuk seluruh TAVP Stack. Berisi:

- **Apa itu TAVP**: Tailwind + Alpine + Volt + Phalcon. Alternatif TALL stack yang lebih cepat.
- **Siapa pemakai**: Developer umum + non-programmer yang dibantu AI.
- **Tech Stack Detail**: Phalcon 5.x, Volt, Tailwind 3.x, Alpine.js 3.x, MySQL/PostgreSQL/SQLite.
- **Architecture**: Request lifecycle, DI container, service providers.
- **Feature List**: Semua fitur yang direncanakan dari 0.1.0 sampai 1.0.0.
- **Milestones**: 10 milestone dari Genesis (0.1.0) sampai Stable Release (1.0.0).
- **Non-Goals**: Apa yang TIDAK dibangun (htmx, Livewire-style, dll).
- **Execution Rules**: Constraint untuk AI executor — jangan invent fitur, schema = suci, frontend pakai CSS variables.

**Audience**: AI executor, developer yang mau kontribusi.

## 2. `prd-cms.html` — PRD Skeleton CMS

PRD untuk TAVP Skeleton CMS — CMS yang mesinnya tetap, body bisa diganti.

- **Konsep**: Chassis mobil. Mesin (admin, registry, theme, renderer) = tetap. Body (section, style, warna) = bisa diganti.
- **Problem Statement**: Setiap project web profil mulai dari nol. Harusnya cukup drag-drop section dari admin.
- **Tech Stack**: Phalcon + Volt + Tailwind (admin only) + Alpine.js + Lucide icons + MySQL.
- **Database Schema**: 9 tabel (layouts, pages, sections, section_content, section_settings, settings, media, contact_messages). Schema = suci, tidak boleh diubah.
- **Section Registry**: 10 section types (hero, about, services, gallery, video, cta, stats, testimonials, contact, faq). Masing-masing punya styles[], fields[], settings[].
- **Theme Engine**: CSS variables generator. 5 warna × 8 opacity variants = 40 CSS variables. 6 preset warna.
- **Admin Panel**: Login, Dashboard, Theme Editor, Page Builder (3 kolom: library, canvas, editor), Pages List.
- **Frontend Renderer**: Loop sections → render partial per type → inject CSS variables. 1 file = multi style.
- **Milestone v0.1.0**: Skeleton bisa dipakai bikin website sederhana dari admin panel tanpa coding.
- **Aturan Eksekusi**: 10 aturan wajib — jangan invent fitur, schema = suci, registry = suci, frontend pakai CSS variables, 1 file = multi style.

**Audience**: AI executor yang mau bangun CMS.

## 4. `coding-standards.html` — Human Style Coding Standards

Standar penamaan dan gaya kode untuk semua repo TAVP.

- **Philosophy**: Kode harus dibaca manusia, bukan mesin. Nama utuh, nggak disingkat.
- **File Naming**: `snake_case.php` untuk model, `PascalCase.php` untuk class, `kebab-case.js` untuk frontend.
- **Variables & Functions**: `snake_case` untuk PHP, `camelCase` untuk JS. Nama deskriptif, bukan abbreviasi.
- **Classes**: `PascalCase`. Nama lengkap, jangan disingkat (UserRepository, bukan UserRepo).
- **Database**: `snake_case` untuk tabel dan kolom. Foreign key = `table_id`.
- **Routes**: `kebab-case` untuk URL (`/user-profile`, bukan `/userProfile`).
- **Config**: `snake_case` untuk key. Group per fungsi.
- **Errors**: Pesan jelas, actionable. Jangan cuma "Error".
- **Comments**: Jelaskan intent, bukan apa yang dilakukan. Hindari comment berlebihan.
- **Anti-Patterns**: DILARANG — `// Generated by AI`, over-abstraction, boilerplate kosong, nama disingkat.

**Audience**: Semua AI executor dan developer yang kontribusi ke TAVP.

## 5. `milestone-checklist.html` — Interactive Milestone Checklist

Checklist interaktif untuk tracking progress semua milestone.

- **10 Milestones**: 0.1.0 (Genesis) → 0.2.0 (Tools) → 0.3.0 (Ecosystem) → 0.4.0 (Polish) → 0.5.0 (Coil) → 0.6.0 (Relay) → 0.7.0 (Intelligence) → 0.8.0 (Marketplace) → 0.9.0 (Maturity) → 1.0.0 (Stable Release).
- **Workstreams per Milestone**: Setiap milestone punya beberapa workstream (Project Setup, Bootstrap, Routing, Controller, Volt, ORM, Migration, Middleware, CLI, Env Adapters, Deploy, Git Abstraction, Auth, Frontend, Templates, Docs).
- **Task IDs**: Setiap task punya ID unik (SET-001, BOOT-001, RT-001, CTL-001, VLT-001, ORM-001, MIG-001, MW-001, CLI-001, ENV-001, DEP-001, GIT-001, AUTH-001, FE-001, TPL-001, DOC-001).
- **Total Tasks**: 138+ tasks di milestone 0.1.0 saja. Lebih banyak di milestone berikutnya.
- **Interactive**: Checkbox tersimpan di localStorage. Progress bar per milestone dan global.
- **Dependencies**: Task ada yang depend ke task lain (terlihat di badge `dep`).

**Audience**: AI executor, project manager, founder.
